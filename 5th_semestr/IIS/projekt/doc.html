Implementace

Celá aplikace je rozdělena na 2 části. Tyto dvě části byly nakonec spojeny do jedné, aby vznikla jedna spustitelná aplikace, kde "index.html" je rendrován jako tzv. template do Django aplikace.

Django

Pro správné fungování aplikace využíváme REST API. V "urls.py" rozdělujeme přichozí požadavky na tzv. endpointy, které se starají o jejich obsloužení. Tyto endpointy jsou implementovány pomocí funkcí v souborech "*views.py", kde tyto funkce komunikují s databází pomocí modelů a serializérů a následně vrací vyžadovaná data v JSON formátu. Tyto data jsou dále zpracována v React aplikaci.

React

Tato frontendová aplikace komunikuje s api a zobrazuje přichozí JSON data v uživatelsky přívětivé podobě a umožňuje také měnit stav databáze pomocí grafického uživatelského rozhraní. Pro vytváření požadavků na api využíváme modulu axios, který komunikaci výrazně zjednodušuje.

Cloudinary

Jelikož heroku nepodporuje ukládání souborů přímo do souborů, tak využíváme služby Cloudinary, která se stará o ukládání souborů(v našem případě fotek).

JWT tokeny

Pro ověřování uživatelů vyuzíváme technologie JWT.

Databáze

Instalace

  • pro hosting jsme si vybrali heroku, kde se postup instalace určuje pomocí souborů: "Procfile", "requirements.txt", "runtime.txt"
  • softwarové požadavky se nachází v souboru "requirements.txt" a "package.json"
  • pro databázový systém jsme zvolili Heroku Postgres, databáze se tvoří automaticky na základě modelů v Django aplikaci "api/models.py". Komunikace s databází je implementována pomocí serializérů v "api/serializers.py" a funkcí v souborech "*views.py".

Známé problémy

Během testování aplikace jsme na žádné problémy nenarazili.
